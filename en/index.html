Monteur

A localized CI software robot to manage your software repository locally and integrable with latest CI service providers. It is built to do more with less noises; business sense.

So Why Use Monteur?

Since there are so many continuous integration services in the market, here are some of the great reasons to use Monteur for your existing and new automation projects:

Avoid Locked-In

Montuer acts as the single unified interface for your CI, let other vendor locked-in CI to interface with Monteur.

Config File Driven

Keep every configuration data into proper data file for maximum consistency CI operations.

Kaizen Spirit

Monteur carries ZORALab’s spirit of Kaizen (改善) - being continuously improved one step at at time.

Cross-Platform = OK

Montuer integrates key actions natively, allowing highest portability possible across various OS and CPU.

Keeping It Simple

Monteur strives to keep automation, commands, way of use documented and simple. Let’s keep it simple.

Localized CI

Monteur allows you to run your CI build fully in local system, enables you to test your CI before any commit.

Open-Source License

Monteur is licensed under OSI’s Apache 2.0 license so you’re always in control: from source to compiled app.

Portability First
