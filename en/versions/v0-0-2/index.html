Version v0.0.2

Monteur released its version v0.0.2 with a bunch of new changes, notable releases and supports, and upgrades.

This release primarily interate major feature improvements over the first prototype to standardize its internal operations and external interactions. There are non-backward compatible changes so please go through this document clearly for what has changed.

Heads Up

There is a bug where the Setup API failed to create the main MonteurFS directories like .monteurFS/bin and .monteurFS/config/config.d. Please upgrade to Monteur version v0.0.3 or manually create those main directories for Monteur to work.

Supported Variants

Here are the list of supported Monteur variants available for different platforms and CPU architectures.

  • linux-amd64 — .deb, .tar.gz

  • linux-arm64 — .deb, .tar.gz

  • linux-arm (v7) — .deb, .tar.gz

  • darwin-amd64 — .tar.gz

  • darwin-arm64 — .tar.gz

  • windows-amd64 — .zip

  • windows-arm64 — .zip

Notable Changes

Here are some of the notable changes for this version.

Backward Compatible

Among the backward-compatible changes are:

  1. Added Init API - Allows Monteur to easily initialize a repository with its configurations. Supported features are:

    • Seamless consistent initialization.

  2. Added Prepare API - Allows Monteur to perform repository updates prior to Test, Build, or Compose API that are otherwise unsuitable to be in any of them. Supported features are:

    • Supported Command.

  3. Added native .tar.gz functions - Allows Monteur to pack and unpack the format without depending on external software.

  4. Added native .zip functions - Allows Monteur to pack and unpack the format without depending on external software.

  5. Added native sha256 functions - Allows Monteur to perform integrity checksum without depending on external software.

  6. Added native sha512 functions - Allows Monteur to perform integrity checksum without depending on external software.

  7. Added native sha512->sha256 functions - Allows Monteur to perform integrity checksum without depending on external software.

  8. Fixed delete-quiet not deleting anything bug - The CEU is now deleting something.

  9. Added secret data support and logger filtration - To protect the confidentiality of the data with dumb-redaction algorithm.

  10. Fixed Concurrency bug for Package API - The Package API will no longer fail in an unpredictable way compared to all other APIs.

  11. Added support for linux/arm platform - Monteur should be able to opreate in the said operating system and CPU.

  12. Added support for darwin/amd64 platform - Monteur should be able to opreate in the said operating system and CPU.

Non-Backward Compatible

Among the backward-compatible changes are:

  1. Renames all jobs directories (e.g. compose/composers, publish/publishers) into an unified name called jobs (e.g. compose/jobs, publish/jobs) - For simpler communications and documentations.
    Actions Required from you:

    1. Manually rename all the said directories accordingly.

  2. Shifted Build API's job directory from app/variants to build/jobs - For simpler communications and documentations and consistency over CI API.
    Actions Required from you:

    1. Manually move and rename all the said directories accordingly.

What's Next

Now that Monteur main features are up and running, the next phase is to ensure maximum compatibility between all release distributions and packaging functions. The following new changes are seen as important and given priority:

  1. Native support of .deb packaging mechanism (complies to linting, security, and standards).

  2. External support of Homebrew distribution for package API and release API.

  3. External support of AppImage distribution for package API and release API.
